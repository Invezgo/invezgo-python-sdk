# Catatan .gitignore

Dokumen ini menjelaskan file dan folder yang di-ignore oleh git.

## File Build & Distribution (TIDAK di-upload ke GitHub)

File-file berikut akan diabaikan karena di-generate saat build dan publish ke PyPI:

- `build/` - Folder build temporary
- `dist/` - Folder distribusi (berisi .whl dan .tar.gz)
- `*.egg-info/` - Egg info directories
- `*.egg` - Egg files
- `*.whl` - Wheel files
- `*.tar.gz` - Source distribution (kecuali jika memang source code)
- `MANIFEST` - Generated manifest file (MANIFEST.in tetap di-track)

## File Development (TIDAK di-upload)

- `.pytest_cache/` - Pytest cache
- `.coverage` - Coverage reports
- `htmlcov/` - HTML coverage reports
- `.tox/` - Tox test environments
- `.mypy_cache/` - MyPy cache
- `venv/`, `.venv/`, `env/` - Virtual environments

## File Secrets & Sensitive (TIDAK di-upload)

- `.env` - Environment variables
- `.env.local` - Local environment
- `*.key`, `*.pem` - Private keys
- `api_keys.txt`, `secrets.txt` - Files yang mungkin berisi secrets

## File Temporary & Backup

- `*.bak`, `*.backup` - Backup files
- `*.tmp` - Temporary files
- `tmp/`, `temp/` - Temporary directories
- `*~` - Editor backup files

## File yang TETAP di-track (Penting!)

✅ File-file berikut **harus** tetap di GitHub:
- `setup.py`
- `pyproject.toml`
- `requirements.txt`
- `MANIFEST.in`
- `README.md`
- `LICENSE`
- Semua file source code di `invezgo/`
- `examples/` folder
- `.gitignore` (file ini)
- `.gitattributes`

## Tips

1. **Setelah build:** Pastikan `dist/` tidak ter-commit
   ```bash
   git status  # Cek apakah ada file di dist/
   ```

2. **Sebelum commit:** Pastikan tidak ada secrets
   ```bash
   git diff  # Review perubahan sebelum commit
   ```

3. **File API spec:** File `invezgo-api*.json` saat ini masih di-track.
   Jika ingin ignore backup copies, uncomment di .gitignore
